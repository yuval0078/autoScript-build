# Quick Start

## 1) Install on another PC (recommended)

1) Download the latest `TouchpadExperimentManager-portable.zip` from GitHub Releases.
2) Unzip anywhere (avoid very long paths).
3) Run `TouchpadExperimentManager.exe`.

Notes:

- Experiments are self-contained ZIPs (config JSON at ZIP root + `audio/*.wav`).
- The app stores its working files under `%APPDATA%\TouchpadExperimentManager`.

## 2) Create + run an experiment (GUI)

1) **Create a New Experiment**
   - Select recordings from any folder on the PC
   - Label words + configure properties
   - Export an experiment ZIP
2) **Run Experiment**
   - Select the exported experiment ZIP
   - The app extracts it and runs the experiment using the ZIP audio
3) **Analyze Results**
   - Open the analyzer from the main menu

## 3) Run from source (developer)

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python main_interface.py
```

## 4) Publish a release (so updates work)

The in-app updater works for the packaged EXE build and uses `update_config.json` + `releases/<channel>/version.json` + `releases/<channel>/version.sha256`.

1) Build the portable ZIP:

```powershell
powershell -ExecutionPolicy Bypass -File build_pyinstaller.ps1 -Version v0.1.0-beta1 -Channel beta -RepoOwner yuval0078 -RepoName autoScript-build
```

2) Commit + push the updated metadata generated by the build:

- `releases/<channel>/version.json`
- `releases/<channel>/version.sha256`

3) Create a GitHub Release where:

- Release tag name == the same `-Version` you built (example: `v0.1.0-beta1`)
- Upload asset file named exactly `TouchpadExperimentManager-portable.zip`
